Graph Representation



Adjacency List: Adjacency lists can be used to represent graphs without multiple edges. The lists specify the vertices that are adjacent to each vertex of the graph.




Fig : A simple graph
and its adjacency list





Directed graphs can also have adjacency lists, where we specify the initial and terminal vertices. In a sparse graph, adjacency lists are much more space-efficient. Adjacency lists do not require a lot of space. However, if we want to identify any one specific edge connection, the worst case scenario is we are checking all entries of that list.




Fig : A directed graph
and its adjacency list

An adjacency matrix has n2 entries. If a simple graph is dense, such as a graph that contains more than half of all possible edges, using an adjacency matrix to represent the graph is usually preferable over using adjacency lists. Using an adjacency matrix, we can determine whether this edge is present by examining the (i, j)th entry in the matrix. This entry is 1 if the graph contains this edge and is 0 otherwise. We need make only one comparison, namely, comparing this entry with 0, to determine whether this edge is present.
